# テキスト作成ガイドライン（Irodori-Studio / Irodori-TTS）

本書は、**読み上げ用テキスト**と、VoiceDesign 利用時の**キャプション**を書くときの指針です。公式の仕様・制約に沿いつつ、制作しやすいように整理しています。

参考: [Irodori-TTS（GitHub）](https://github.com/Aratako/Irodori-TTS)、[VoiceDesign モデルカード](https://huggingface.co/Aratako/Irodori-TTS-500M-v2-VoiceDesign)

---

## 1. まずモードを決める

| モード | 使いどころ | 話者の決め方 |
| -------- | ------------ | ---------------- |
| **参照音声（基本モデル）** | 特定の声に**近づけたい**とき | 参照 WAV（ゼロショット）／参照なし |
| **VoiceDesign** | **参照なし**で、説明文だけで声質・演技を決めたいとき | **キャプション**（＋本文の絵文字） |

同じ一文でも、モードで「話者の決まり方」が違うので、先にモードを決めてから本文を書くと迷いが減ります。

---

## 2. 読み上げ本文（全モード共通）

### 2.1 言語

- **日本語**が前提です（モデル仕様）。
- 英語や記号だけの長文は、想定外の抑揚になることがあります。

### 2.2 漢字・読み

- 公式の注意として、**難読漢字・固有名詞**は、誤読されやすいです。
- 重要な名前や読みを固定したいときは、**ひらがな・カタカナに開く**、**ルビを振る代わりにかな表記**する、などの対策が確実です。

### 2.3 長さと区切り

- **1行が長すぎる**と、息継ぎや区切りが不自然になりやすいです。会話なら**発話ごとに改行**、地の文なら**句点で区切る**と調整しやすいです。
- セリフの「間」は、**改行**や**記号（…… など）**で雰囲気を出せます（過剰にしすぎない）。

### 2.4 絵文字（感情・効果）

- 本文に**絵文字を入れる**と、感情や効果のニュアンスを足しやすいです（VoiceDesign ではキャプションと**併用**される想定）。
- 一覧・意味の参考: モデル付属の [EMOJI_ANNOTATIONS.md](https://huggingface.co/Aratako/Irodori-TTS-500M-v2-VoiceDesign/blob/main/EMOJI_ANNOTATIONS.md)（Hugging Face 上）。
- **絵文字だらけ**にすると、ラジオ風・コメディ寄りになりやすい、とコミュニティ記事でも言及されています。**足したいところにだけ**使うのが無難です。

### 2.5 記号・構文

- 括弧 `（）` での舞台指示は、モデルによっては**読まれ方が安定しない**ことがあります。必須なら**短く**、または**地の文に直す**選択も検討してください。
- 数字・英字は**読みを意識**（「2026」→「にせんにじゅうろく」など、必要ならかなに）。

## 3. VoiceDesign 用キャプション

キャプションは **「誰が・どんな声で・どんな感情で・どんな距離感で」** を**日本語の散文**で書きます。参照音声は使いません。

### 3.1 書くとよい要素（例）

- **声の高さ**（低め／高め／中年 など）
- **性別・年齢のイメージ**（若い女性、少年、老人 など）
- **感情・態度**（苛立ち、申し訳なさ、威圧、ささやき など）
- **距離感**（マイクに近い、遠くで叫んでいる など）
- **話し方**（早口、ゆっくり、ナレーション調 など）

### 3.2 公式・デモで示されている例

モデルカードでは、**同じ本文**に対して、**キャプションを変える**と聞き取りの印象が変わることが示されています。  
「情景が伝わる**具体的な指示**」の方が、狙いに寄りやすい傾向です。

### 3.3 注意

- **矛盾した指示**（とても小声で、全員に聞こえる怒鳴り）などは、安定しにくいことがあります。
- **実在の特定人物の声の再現**は、ライセンス・倫理上の注意が公式に書かれています。創作の「タイプ表現」に留めるのが安全です。

---

## 4. 参照音声モード（基本モデル）向け

- **参照 WAV** は、可能なら**ノイズが少なく**、**話者がはっきりした区間**を使うと安定しやすいです。
- **参照なし**は、話者固定より**汎用読み上げ**に近い結果になりやすいです。

---

## 5. 制作の流れ（おすすめ）

1. **モード**を選ぶ（参照 or VoiceDesign）。
2. **本文**を書く（必要ならプリセットから流用・改稿）。
3. VoiceDesign なら **キャプション**を書く（またはキャラプリセットで仮入れしてから微調整）。
4. 必要なら本文に **絵文字**を追加。
5. 生成して聞き、**キャプション・本文・絵文字**のどこを変えるか決める。

---

## 6. Irodori-Studio 内の補助機能

- **保存プロファイル**: 自分で保存した設定・本文・キャプション・参照設定などを丸ごと再利用。
- **台本サンプル**: 雰囲気の違う例文をすばやく差し替え。
- **VoiceDesign キャラプリセット**: キャプションの出発点として利用。
- **ファイル名自動**: モードと本文先頭から保存名を生成（整理・検索向け）。
- **ヘルプのテキスト作成ガイド**: アプリ内からこの文書を開けます。
